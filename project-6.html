Overview

A structured collection of diagrams, patterns, and written notes capturing my learning journey in system design.

The Problem

System design concepts are often scattered across videos, articles, and talks, making it hard to build a coherent mental model.

The Solution

I created a dedicated “System Design Notes” project: a living knowledge base where I organize concepts, patterns, and example architectures in a way that I can revisit and extend over time.

Contents

  • Core concepts (latency, throughput, availability, consistency)
  • Common patterns (caching, load balancing, sharding, queues)
  • Example architectures (URL shortener, messaging app, feed system)
  • Trade‑off discussions and design decisions
  • Hand‑drawn or digital diagrams

Format & Tools

  • Markdown documents for written notes
  • Diagrams (stored in assets/media/)
  • Optional: simple HTML pages to browse topics
  • Version control with Git to track how my understanding evolves

What I Learned

This project helped me move from “watching system design content” to actually internalizing it—by rewriting concepts in my own words and sketching architectures from scratch.
